## Conventions

<!-- cartographer:section id="conventions" -->

- All git/filesystem failures degrade gracefully (empty results), never crash — churn and entry points are optional signals.
- Paths are normalized to POSIX everywhere via `toPosix`; module identity is its POSIX-relative path.
- One file per concern; parsers in `deps.ts` are pluggable objects (`test` regex + `extract` fn).

## Architecture notes

<!-- cartographer:section id="architecture" -->
- Pipeline: `scan` (modules + hashes) → `churn`/`deps`/`entrypoints` (independent signals) → `emit` (merge + render) → `index` (CLI).
- MAP.md is both output and state: `emit.parseExisting` reads agent descriptions back and preserves them when module hashes match.
- Staleness = content-hash mismatch per module; stale markers quote the previous description so agents update with context.
- The LLM layer is deliberately external: the CLI never calls an API — the consuming agent fills marker-anchored placeholders.
